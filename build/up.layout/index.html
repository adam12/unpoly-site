Viewport scrolling

This modules contains functions to scroll the viewport and reveal contained elements.

Unobtrusive behavior
Elements matching these CSS selectors will automatically gain useful behavior when they are added to the DOM tree.
[up-viewport]

Marks this element as a scrolling container. Apply this ttribute if your app uses a custom panel layout with fixed positioning instead of scrolling <body>.

up.reveal will always try to scroll the viewport closest to the element that is being revealed. By default this is the <body> element.

Example

Here is an example for a layout for an e-mail client, showing a list of e-mails on the left side and the e-mail text on the right side:

.side {
  position: fixed;
  top: 0;
  bottom: 0;
  left: 0;
  width: 100px;
  overflow-y: scroll;
}

.main {
  position: fixed;
  top: 0;
  bottom: 0;
  left: 100px;
  right: 0;
  overflow-y: scroll;
}

This would be the HTML (notice the up-viewport attribute):

<div class=".side" up-viewport>
  <a href="/emails/5001" up-target=".main">Re: Your invoice</a>
  <a href="/emails/2023" up-target=".main">Quote for services</a>
  <a href="/emails/9002" up-target=".main">Fwd: Room reservation</a>
</div>

<div class="main" up-viewport>
  <h1>Re: Your Invoice</h1>
  <p>
    Lorem ipsum dolor sit amet, consetetur sadipscing elitr.
    Stet clita kasd gubergren, no sea takimata sanctus est.
  </p>
</div>
[up-fixed=top]

Marks this element as a navigation fixed to the top edge of the screen using position: fixed.

up.reveal is aware of fixed elements and will scroll the viewport far enough so the revealed element is fully visible.

Example:

<div class="top-nav" up-fixed="top">...</div>
[up-fixed=bottom]

Marks this element as a navigation fixed to the bottom edge of the screen using position: fixed.

up.reveal is aware of fixed elements and will scroll the viewport far enough so the revealed element is fully visible.

Example:

<div class="bottom-nav" up-fixed="bottom">...</div>
Javascript methods
Use these methods to programmatically interact with Up.js from your own Javascript code.
up.layout.defaults([options])
Parameters
[options.viewport] String optional
[options.fixedTop] String optional
[options.fixedBottom] String optional
[options.duration] Number optional
[options.easing] String optional
[options.snap] Number optional
up.scroll(viewport, scrollPos, [options]) protected

Scrolls the given viewport to the given Y-position.

A "viewport" is an element that has scrollbars, e.g. <body> or a container with overflow-x: scroll.

Example

This will scroll a <div class="main">...</div> to a Y-position of 100 pixels:

up.scoll('.main', 100);

Animating the scrolling motion

The scrolling can (optionally) be animated.

up.scoll('.main', 100, {
  easing: 'swing',
  duration: 250
});

If the given viewport is already in a scroll animation when up.scroll is called a second time, the previous animation will instantly jump to the last frame before the next animation is started.

Parameters
viewport String or Element or jQuery

The container element to scroll.

scrollPos Number

The absolute number of pixels to set the scroll position to.

[options.duration] Number optional

The number of miliseconds for the scrolling's animation.

[options.easing] String optional

The timing function that controls the acceleration for the scrolling's animation.

Return value
Deferred

A promise that will be resolved when the scrolling ends.

up.reveal(element, [options])

Scroll's the given element's viewport so the element is visible for the user.

By default Up.js will always reveal an element before updating it with Javascript functions like up.replace or UJS behavior like [up-target].

How Up.js finds the viewport

The viewport (the container that is going to be scrolled) is the closest parent of the element that is either:

  • the currently open modal
  • an element with the attribute [up-viewport]
  • the <body> element
  • an element matching the selector you have configured using up.viewport.defaults({ viewSelector: 'my-custom-selector' })

Fixed elements obstruction the viewport

Many applications have a navigation bar fixed to the top or bottom, obstructing the view on an element.

To make up.aware of these fixed elements you can either:

  • give the element an attribute up-fixed="top" or up-fixed="bottom"
  • configure default options for fixedTop or fixedBottom
Parameters
element String or Element or jQuery
[options.viewport] String or Element or jQuery optional
[options.duration] Number optional
[options.easing] String optional
[options.snap] String optional
Return value
Deferred

A promise that will be resolved when the element is revealed.
