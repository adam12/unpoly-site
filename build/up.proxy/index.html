Caching and preloading

All HTTP requests go through the Up.js proxy. It caches a limited number of server responses for a limited amount of time, making requests to these URLs return insantly.

The cache is cleared whenever the user makes a non-GET request (like POST, PUT or DELETE).

The proxy can also used to speed up reaction times by preloading links when the user hovers over the click area (or puts the mouse/finger down before releasing). This way the response will already be cached when the user performs the click.

Unobtrusive behavior
Elements matching these CSS selectors will automatically gain useful behavior when they are added to the DOM tree.
[up-preload]

Links with an up-preload attribute will silently fetch their target when the user hovers over the click area, or when the user puts her mouse/finger down (before releasing). This way the response will already be cached when the user performs the click, making the interaction feel instant.

Parameters
[[up-delay]=50] optional

The number of milliseconds to wait between hovering and preloading. Increasing this will lower the load in your server, but will also make the interaction feel less instant.

Javascript methods
Use these methods to programmatically interact with Up.js from your own Javascript code.
up.proxy.defaults([options])
Parameters
[options.preloadDelay] Number optional
[options.cacheSize] Number optional
[options.cacheExpiry] Number optional

The number of milliseconds until a cache entry expires.

up.proxy.ajax(request.url, [request.method], [request.selector])

Makes a request to the given URL and caches the response. If the response was already cached, returns the HTML instantly.

If requesting a URL that is not read-only, the response will not be cached and the entire cache will be cleared. Only requests with a method of GET, OPTIONS and HEAD are considered to be read-only.

Parameters
request.url String
[request.method='GET'] String optional
[request.selector] String optional
